# 小光 UI 与宠物建模优化设计

## 目标

将默认宠物“小光”升级为圆润的暖光小精灵，并把当前隐蔽的边缘侧栏改为“默认陪伴、按需展开”的轻量交互。改版必须保留既有聊天、笔记、提醒、搜索、翻译、天气、设置、拖拽、双击和长按能力，以及所有内置与自定义皮肤。

## 视觉语言

- **小光**：无动物属性的小精灵。轮廓由圆润头部、较小躯干与柔和漂浮感组成；主色为暖金至珊瑚渐变，深靛蓝五官，低强度呼吸微光。
- **情绪表达**：通过眼睛高光、眼睑、嘴形、腮红和光粒变化表达开心、兴奋、思考、害羞、无聊与困倦；不使用高频或大面积闪烁。
- **界面**：透明深靛蓝为底，暖金用于强调，珊瑚仅用于宠物与少数状态反馈。统一为低对比半透明面板、18px 左右圆角、清晰焦点环和可读的文字对比度。

## 交互与信息层级

### 收起态

默认只渲染小光与紧凑心情胶囊，不显示工具按钮。这样桌面保持干净，宠物仍可点击、双击、长按或拖拽。

### 展开态

单击小光时，在宠物右侧出现轻量快捷面板，包含问候、当前状态和六个已有工具入口：笔记、提醒、搜索、翻译、天气、设置。再次单击宠物、点击面板关闭控件或使用 Escape 收起面板。工具入口打开现有面板；聊天入口继续显示既有对话气泡。

双击继续触发“玩耍/兴奋”，长按继续触发害羞反馈，拖拽阈值与窗口拖动保持不变。面板打开后不拦截宠物本体的拖拽事件。

## 实现边界

1. **宠物精灵生成**：在现有 `SpriteGenerator`/`PixelRenderer` 流程中重绘 `lumie` 帧。维持现有帧尺寸、缓存键和 `PetTheme` 接口；其他皮肤不改造。
2. **动画**：为 idle、walk/run、play、sleep、thinking、shy 等既有动作提供连贯帧。发光与粒子保持 Canvas 后处理，避免把特效烘焙进所有帧。
3. **UI 状态**：在 Zustand store 中增加明确的 `isQuickPanelOpen` 和操作方法；`QuickMenu` 从 hover-only 侧栏变为受控的快捷面板。现有 `activePanel` 仍只负责完整工具面板。
4. **组合层**：`App` 使用收起/展开状态决定快捷面板、心情胶囊与聊天的位置；不改变 hooks 对提醒、聊天、渲染器和交互物理的职责。
5. **样式**：将通用颜色、阴影、模糊和动画时长收束为 CSS 自定义属性。改造 `App.css` 和 `QuickMenu.css`，同时保持小窗口（300×350）内不裁切、可键盘聚焦。

## 数据流与错误处理

`canvas click → usePetInteraction → petStore.toggleQuickPanel → App/QuickMenu render`。若点击落在拖拽、长按或双击路径上，不切换快捷面板。快捷按钮经现有 `handleQuickAction` 打开对应工具面板；未知动作维持现有安全忽略行为。

精灵生成沿用 IndexedDB 缓存。生成或缓存读取失败时，保留当前加载指示器，并避免渲染半成品帧。

## 验证

- 单元测试：小光帧表与每个动作帧数量；快捷面板开关与工具动作映射；皮肤切换后缓存键和主题不串色。
- 构建验证：`pnpm test:run` 和 `pnpm build`。
- 手动验证：收起/展开、Escape、宠物单击/双击/长按/拖拽、六个入口、聊天、提醒通知、小光和三种内置替换皮肤，以及窄窗口中的无裁切显示。

## 非目标

- 不新增账号、后端、外部素材下载或 3D 引擎。
- 不重写工具功能与 AI 服务。
- 本次不改造非默认皮肤的造型；它们只继承新的 UI 容器和交互方式。
